# Contributing to PDD

Thanks for your interest in contributing to PDD.

## Development Flow

1. Fork or create a branch
2. Make focused changes
3. Use clear commit messages
4. Open a pull request with context and validation notes

## Commit Conventions

PDD uses conventional-style commits:

- `feat:` new functionality
- `fix:` bug fixes
- `docs:` documentation
- `chore:` maintenance
- `refactor:` safe internal improvements

## What Good Contributions Look Like

- minimal safe delta
- clear reasoning
- verification notes
- no unrelated changes

## Pull Request Checklist

- explain the change
- describe risk
- describe validation
- keep scope small

## Ideas to Contribute

- new CLI commands
- better templates
- improved examples
- AI integrations
- docs and tutorials
